# SnPy Code Header Standards
# ------------------------------------------------------------

All new Python scripts must include a code header at the very top of the file, following this exact structure and order. This ensures consistency, traceability, and professionalism across the codebase.

## Header Template

# =========================================================
# SCRIPT TITLE (e.g., PDF TO PNG CONVERTER)
# =========================================================
#
# FILENAME  |  <script_filename.py>
# DIRECTORY |  <absolute_or_relative_path_to_script>
#
# AUTHOR    |  <author_name_or_team>
# DATE      |  <YYYY-MM-DD>
#
# DESCRIPTION
# - Briefly describe the script's purpose and main features.
# - Use bullet points for clarity.
#
# DEVELOPMENT LOG
# <version> - <date> |  <summary of change>
# - List key changes, features, or fixes for each version.
# - Use bullet points for details.
#
# =========================================================

## Field Explanations

- **SCRIPT TITLE**: A short, descriptive title in all caps.
- **FILENAME**: The exact name of the script file.
- **DIRECTORY**: The absolute or relative path to the script file.
- **AUTHOR**: The main author or team responsible for the script.
- **DATE**: The date the script was first created (YYYY-MM-DD).
- **DESCRIPTION**: A concise summary of what the script does, its main features, and any important notes. Use bullet points for readability.
- **DEVELOPMENT LOG**: A chronological log of changes, versions, and key updates. Each entry should include the version, date, and a summary, followed by bullet points for details.

## Notes
- Do not omit or reorder any fields.
- Use all caps for the script title.
- Use consistent indentation and spacing as shown.
- Update the DEVELOPMENT LOG with every significant change.
